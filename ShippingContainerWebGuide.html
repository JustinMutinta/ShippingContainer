1 What Changed from the Desktop Version

The desktop Swing app and the web app solve the same problem, but in completely different ways. The table below shows what was kept, what was replaced, and what is new.

ConcernDesktop (Swing)Web (Spring Boot)
UI renderingJava Swing — OS windowsHTML + CSS in a browser
User interactionActionListeners on buttonsJavaScript Fetch API calls
Data storageIn-memory arrays (lost on exit)SQLite database (persists)
Data modelShipData.java (plain class)Item.java (JPA Entity)
Business logicInside ShipDataShipService.java
CommunicationDirect Java method callsHTTP REST API (/api/...)
LayoutBorderLayout, BoxLayoutCSS Grid and Flexbox

The business rules (slot capacity, add/remove logic) transferred almost unchanged. Only the delivery mechanism changed — from Java method calls to HTTP requests.

3 The Build File: pom.xml

Maven is a build tool for Java. Its configuration file, pom.xml, declares the project's dependencies (libraries it needs) and build settings. Spring Boot's parent POM provides sensible defaults and manages library versions automatically. 

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.5</version>
</parent>

The Four Dependencies

DependencyWhat it provides
spring-boot-starter-web Embedded Tomcat web server + REST support. Without this, there is no HTTP server.
spring-boot-starter-data-jpa Hibernate ORM + Spring Data — lets you query a database using Java objects instead of SQL.
sqlite-jdbc The JDBC driver that lets Java talk to an SQLite file.
hibernate-community-dialects Teaches Hibernate the quirks of SQLite (e.g. how auto-increment IDs work).
Why an embedded server?
Spring Boot bundles Tomcat inside the JAR file. You do not need to install a separate web server — running mvn spring-boot:run is enough to get a fully working HTTP server on port 8080.

Running the App

# From the ShippingContainerWeb/ directory:
mvn spring-boot:run

Then open http://localhost:8080 in any browser.

4 The Database: SQLite + JPA

Why SQLite?

SQLite stores an entire database in a single file (shipping.db). There is no server to install or configure. When the Spring Boot app starts, it opens that file (creating it if it does not exist). This makes it ideal for small, self-contained applications.

application.properties

spring.datasource.url=jdbc:sqlite:shipping.db
spring.datasource.driver-class-name=org.sqlite.JDBC
spring.jpa.properties.hibernate.dialect=org.hibernate.community.dialect.SQLiteDialect
spring.jpa.hibernate.ddl-auto=update
PropertyWhat it does
datasource.urlPoints to the SQLite file. A relative path puts it in the working directory.
driver-class-nameTells Spring which JDBC driver to use to connect.
hibernate.dialectTells Hibernate which SQL "dialect" to generate — SQLite has some differences from MySQL or PostgreSQL.
ddl-auto=updateHibernate creates the items table automatically on first run, and updates it if the schema changes.

What JPA Does

JPA (Java Persistence API) is a standard for mapping Java objects to database rows. Hibernate is the implementation of that standard used here. Instead of writing: 

// Raw SQL — error-prone and verbose
ResultSet rs = stmt.executeQuery("SELECT * FROM items WHERE container_name IS NULL");

You write:

// JPA — type-safe, no SQL
List<Item> available = itemRepository.findByContainerNameIsNull();

Hibernate translates the method name into the correct SQL query automatically.

5 The Data Layer: Entity and Repository

Item.java — The Entity

An entity is a Java class that maps directly to a database table. Each field becomes a column; each instance becomes a row. 

@Entity
@Table(name = "items")
public class Item {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;             // auto-assigned by SQLite

    private String  description;
    private String  serialNumber;
    private int     height, width, length, weight;

    @Column(name = "container_name")
    private String  containerName; // null = available; "Container 1" = loaded

    @Column(name = "slot_index")
    private Integer slotIndex;     // which slot within the container
}

The two nullable fields — containerName and slotIndex — replace the entire array structure from the desktop version. A null containerName means the item is available; a non-null value means it is loaded into that container at that slot. One table, one query, same data.

Annotation Quick Reference

AnnotationMeaning
@EntityMarks this class as a JPA entity — Hibernate will manage it.
@Table(name=...)Sets the database table name (defaults to the class name if omitted).
@IdThis field is the primary key.
@GeneratedValueThe database auto-generates this value (SQLite uses ROWID).
@Column(name=...)Maps the field to a column with a specific name.

ItemRepository.java — The Repository

A repository is an interface that Spring Data implements for you at runtime. Declaring a method with the right name is all that is needed — no SQL, no implementation class. 

public interface ItemRepository extends JpaRepository<Item, Long> {

    List<Item> findByContainerNameIsNull();          // SELECT * WHERE container_name IS NULL
    List<Item> findByContainerName(String name);    // SELECT * WHERE container_name = ?
}

JpaRepository<Item, Long> also provides built-in methods for free: findById(), save(), count(), saveAll(), and more.

Spring Data method naming convention
Spring Data reads the method name and generates the query. findBy → SELECT, ContainerName → WHERE container_name, IsNull → IS NULL. Chaining words builds the WHERE clause.

6 The Business Logic: ShipService

The service layer sits between the controller (web) and the repository (database). It owns all the business rules: slot capacity, finding the first empty slot, and deciding whether an add or remove is valid. 

@Service
@Transactional
public class ShipService {

    @Autowired
    private ItemRepository itemRepository;

    public static final Map<String, Integer> CAPACITIES = Map.of(
            "Container 1", 5,  "Container 2", 5,  "Container 3", 5,
            "Platform 1",  1,  "Platform 2",  1
    );
}

@Service and @Transactional

  • @Service — marks this as a Spring-managed bean. Spring creates one instance and injects it wherever it is needed.
  • @Transactional — wraps every method in a database transaction. If anything goes wrong mid-method, all changes are rolled back automatically.
  • @Autowired — Spring injects the ItemRepository instance automatically. You never call new ItemRepository().

The Add Logic

public boolean addToContainer(Long itemId, String containerName) {
    Item item = itemRepository.findById(itemId).orElse(null);

    // Guard: item must exist and not already be in a container
    if (item == null || item.getContainerName() != null) return false;

    Integer capacity = CAPACITIES.get(containerName);
    List<Item> current = itemRepository.findByContainerName(containerName);

    // Guard: container must have space
    if (current.size() >= capacity) return false;

    // Find first slot not already occupied
    Set<Integer> usedSlots = current.stream()
            .map(Item::getSlotIndex).filter(Objects::nonNull)
            .collect(Collectors.toSet());

    for (int i = 0; i < capacity; i++) {
        if (!usedSlots.contains(i)) {
            item.setContainerName(containerName);
            item.setSlotIndex(i);
            itemRepository.save(item);   // writes to SQLite
            return true;
        }
    }
    return false;
}

The remove logic is simpler: find the item by ID, set both containerName and slotIndex back to null, and save. The item immediately reappears in the available pool.

7 The REST API: ShipController

A REST API exposes data and operations over HTTP. The browser sends HTTP requests; the server responds with JSON. The controller maps each URL to a Java method. 

@RestController
@RequestMapping("/api")
public class ShipController {

    @Autowired
    private ShipService shipService;

    @GetMapping("/items/available")
    public List<Item> getAvailableItems() {
        return shipService.getAvailableItems();
    }

    @PostMapping("/containers/{name}/items/{itemId}")
    public ResponseEntity<ApiResponse> addItem(
            @PathVariable String name,
            @PathVariable Long   itemId) {
        boolean success = shipService.addToContainer(itemId, name);
        if (success)
            return ResponseEntity.ok(new ApiResponse(true,  "Added to " + name));
        return ResponseEntity.badRequest()
                              .body(new ApiResponse(false, name + " is full."));
    }

    @DeleteMapping("/items/{itemId}/placement")
    public ResponseEntity<ApiResponse> removeItem(@PathVariable Long itemId) {
        boolean success = shipService.removeFromContainer(itemId);
        ...
    }
}

The Full API Surface

ResponseEntity and HTTP Status Codes

ResponseEntity lets you control the HTTP status code alongside the response body. A successful add returns 200 OK; a full container returns 400 Bad Request. The browser's JavaScript checks data.success to decide which status message to show the user.

JSON Serialisation

Spring Boot includes Jackson, which automatically converts Java objects to JSON. Returning a List<Item> from a controller method produces:

  { "id": 1, "description": "Box 1", "serialNumber": "No Serial Number",
    "height": 10, "width": 10, "length": 10, "weight": 5,
    "containerName": null, "slotIndex": null },
  ...
]

8 Seeding the Database: DataInitializer

The app needs starter data on first launch. DataInitializer inserts the 13 items into the database automatically — but only if the table is empty, so it does not duplicate data on every restart. 

@Component
public class DataInitializer {

    @Autowired
    private ItemRepository itemRepository;

    @PostConstruct
    public void seed() {
        if (itemRepository.count() > 0) return; // already seeded

        itemRepository.saveAll(List.of(
            new Item("Box 1", "No Serial Number", 10, 10, 10, 5),
            new Item("BMW Vehicle", "45RFT90RV", 60, 60, 120, 450),
            ... // 13 items total
        ));
    }
}
AnnotationWhat it does
@ComponentRegisters this class with Spring so it is managed and its lifecycle hooks run.
@PostConstructRuns this method once, after Spring has finished wiring everything together. At this point the database connection and JPA are fully ready.

9 The Frontend: HTML, CSS, and JavaScript

The entire user interface is three static files served from src/main/resources/static/. The browser downloads them once and then communicates with the server only through the REST API — no page reloads.

The HTML defines the skeleton: a header, a three-column workspace, a status bar, and a details table. Every interactive element has an id so JavaScript can find and update it.

app.js — The Fetch API

JavaScript's Fetch API sends HTTP requests to the Spring Boot server and reads the JSON responses. Using async/await keeps the code readable despite the asynchronous nature of network calls. 

async function addItem() {
    const name = encodeURIComponent(currentContainer); // "Container 1" → "Container%201"
    const res  = await fetch(`/api/containers/${name}/items/${selectedAvailId}`, {
        method: 'POST'
    });
    const data = await res.json();           // parse JSON response
    setStatus(data.message, data.success ? 'success' : 'error');
    if (data.success) await refreshAll();   // re-fetch and re-render both lists
}

State and Re-rendering

The JavaScript keeps three pieces of state:

  • availableItems — the array last fetched from /api/items/available
  • containerMap — a map of container name → contents, fetched from /api/containers
  • selectedAvailId / selectedContId — which item is highlighted in each list

Every add or remove calls refreshAll(), which re-fetches both endpoints in parallel using Promise.all() and then re-renders every panel. This keeps the UI in sync with the database without a page reload. 

async function refreshAll() {
    const [itemsRes, containersRes] = await Promise.all([
        fetch('/api/items/available'),
        fetch('/api/containers')
    ]);
    availableItems = await itemsRes.json();
    const list     = await containersRes.json();
    containerMap = {};
    list.forEach(c => containerMap[c.name] = c);
    renderAvailablePanel();
    renderContainerPanel();
    renderDetailsTable();
}

11 Key Concepts Explained

Dependency Injection

Instead of creating objects with new, Spring creates them and injects them where needed. @Autowired is the signal: "Spring, please give me an instance of this." This means the controller never needs to know how to create a ShipService — Spring handles it.

Layers and Why They Are Separated

LayerKnows aboutDoes not know about
ControllerHTTP, URLs, JSONSQL, business rules
ServiceBusiness rules, entitiesHTTP, HTML
RepositoryDatabase queriesHTTP, business rules

This separation means you could swap SQLite for PostgreSQL by changing only the configuration — the controller and service code would not change at all.

async / await in JavaScript

Network calls take time. JavaScript is single-threaded, so blocking while waiting would freeze the browser. async/await lets you write code that looks synchronous but does not block — the browser remains responsive while waiting for the server to respond.

encodeURIComponent

URLs cannot contain spaces. "Container 1" becomes "Container%201" when encoded. Spring Boot decodes it back to "Container 1" automatically when binding the @PathVariable. Always encode dynamic values before putting them in a URL.

Promise.all

Promise.all([fetchA, fetchB]) fires both HTTP requests at the same time and waits for both to complete before continuing. This halves the wait time compared to calling them one after the other.

Why Persistence Matters

The desktop app lost all cargo state when it closed. With SQLite, the state is written to disk on every save. Restarting the server, closing the browser, or rebooting the machine — the containers still hold the same cargo because it is stored in shipping.db, not in memory.

What to build next
  • Add new items via the UI — a form that POSTs to a new /api/items endpoint, inserting a new row into SQLite.
  • Delete items — a DELETE endpoint and a trash button in the UI.
  • User accounts — Spring Security can protect the API with a login page.
  • Deploy it — run mvn package to produce a self-contained JAR, then deploy to any server with Java installed.
